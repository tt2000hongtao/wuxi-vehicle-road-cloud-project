# 文档资产库目录规范

本文档资产库用于承载无锡车路云一体化项目管理平台的合同、会议纪要、变更单、项目计划、出入库单、安装签证、验收资料、地图配置资产等文件原件与派生文件。前端应用只读取元数据，不直接内嵌大文件。

## 目录结构

```text
document_assets/
├── raw/              # 原始文件库：保留文件原件，不直接修改
├── derived/          # 派生解析结果：文本、表格抽取、结构化字段、质检结果
├── preview/          # 预览文件：缩略图、PDF 预览、图片预览、SVG 预览引用
└── import_batches/   # 批量导入清单、扫描报告、校验报告
```

## 存储口径

- 原件进入 `raw/`，不得由业务流程直接覆盖。
- 每个文件入库时计算 `sha256`，用于去重、版本识别和完整性校验。
- 数据库保存 `storage_key`，不要把本机绝对路径作为唯一依据。
- `storage_key` 推荐格式：

```text
raw/{project_code}/{document_category}/{yyyy}/{sha256前2位}/{sha256}/{原文件名}
```

- 本地阶段 `storage_key` 映射到本目录；云端阶段可映射到 MinIO/S3/OSS/COS 对象键。

## 批量导入原则

- 不删除原始来源文件。
- 扫描、复制、登记、解析、校验分步骤执行。
- 每批导入必须生成批次号、文件数、总大小、重复数、解析成功数、解析失败数、未关联业务对象文件数。
- 地图 MAP/RSI/信号机配置文件继续按“配置资产”治理，不降级为普通附件。
