# 无锡车路云一体化项目管理平台一期产品方案书（修订版）

## 1. 修订依据

本次修订基于以下两份项目数据文件：

- `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/点位管理表-无锡车路云.xlsx`
- `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/设备安装位置表.xlsx`

点位数以有效点位行数为准，不以 Excel 物理行数直接作为点位总数。当前点位管理表可识别有效点位 2706 条；部分规划行尚未确定 NodeID 或 CrossID，应进入待补全数据治理，不计入正式点位数。核心字段包括：序号、点位类型、行政区域、NodeID、CrossID、点位英文名称、CGCS2000 原始规划经纬度、点位名称、信号机厂商、感知点位类型、是否自适应点位、是否可变车道路口、可变车道数量、可变车道方向。业务页面默认仅展示 GCJ-02 坐标，原始 CGCS2000 坐标仅用于原始数据记录和审计。

设备安装位置表包含 `Sheet1` 与 `未匹配` 两个工作表。`Sheet1` 记录已匹配到点位的设备安装信息，核心关联字段为 `NodeID`、`点位名称`、`安装位置`、`物料编码`、`物料名称`、`供应商单位`、`合同名称`、`品牌型号`、`送货/入库/开箱验收/领料/安装数量` 及相关单据索引。`未匹配` 工作表记录未能关联到点位的设备、车端、机房或服务类条目，需作为数据治理对象纳入系统。

运维管理功能补充基于每日路侧设备运行状态表。该表用于形成运维状态快照、异常日历、离线/异常设备点位地图和离线/异常趋势图，不得与设备安装位置表混用。运维状态表中的 `路口ID` 字段用于关联点位管理表中的 `NodeId` 字段，点位坐标以点位管理表坐标为准。

## 2. 产品定位

建设一套面向无锡车路云一体化项目的建设、资产、仓储、合同、运维全过程管理平台。系统以 `NodeID` 点位为项目空间主线，以设备安装位置为资产落位主线，以合同、入库、出库、验收、工单为业务过程主线，形成项目数字化管理闭环。

## 3. 一期建设目标

一期重点实现以下目标：

1. 基于现有点位管理表建立标准点位库，实现一点一档。
2. 基于设备安装位置表建立设备安装与合同履约台账。
3. 点位管理提供表格视图、卡片视图、地图视图三种工作方式。
4. 任一点位均可进入点位编辑页，集中维护基础信息、地图定位、周边点位、设备清单和一点一档资料。
5. 点位编辑页嵌入高德地图，使用 GCJ-02 坐标展示当前点位并自动显示周边 4-5 个最近点位。
6. 系统存储原始 CGCS2000 坐标作为审计数据，并转换为 GCJ-02 坐标用于业务页面、地图展示和距离计算。
7. 设备管理以 `NodeID + 安装位置 + 物料编码/物料名称` 关联点位，支持查看设备履约、入库、领料、安装状态。
8. 将设备表中的未匹配数据纳入待治理池，支持人工匹配到点位、仓库、机房或非点位资产。
9. 增加复杂合同管理能力，支持移动+天安总集结构、多级合同流、设备级合同明细、合同与点位设备穿透关联。
10. 增加运维管理能力，支持路侧设备运行状态表导入、历史状态快照保存、异常日历、离线/异常设备点位地图和离线/异常趋势分析。
11. 增加应用视觉风格切换能力，至少支持默认大屏风格、命令风格和轨迹风格，满足项目演示、运维监控和汇报展示的不同场景。

## 4. 核心业务对象

### 4.0 项目总览入口

总览页卡片作为一期核心功能入口：

- 项目点位卡片点击进入点位管理功能页。

- 未匹配设备卡片点击进入未匹配池功能页，展示 2447 条未匹配记录，支持按物料、供应商、合同板块治理到点位、仓库、机房或服务项。
- 合同流卡片点击进入合同管理功能页；卡片名称统一为“合同流”，内容分为天安前向合同和天安后向合同，卡片名称不使用合同流明细行口径。

总览页同时提供两个导入操作：

- 导入点位管理表：用于导入点位管理表并生成/更新点位库，模板格式参考 `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/点位管理表-无锡车路云.xlsx`。
- 导入设备管理表：用于导入设备点位分布信息并按 NodeID 关联点位，模板格式参考 `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/设备安装位置表.xlsx`。

### 4.1 点位

点位是项目管理的空间载体。系统以 `NodeID` 作为点位主键，`CrossID` 作为路口/交叉口辅助标识。

点位对象应覆盖：

- 路口点位
- 路段点位
- 匝道汇入点位
- 匝道汇出点位
- V2P 点位
- 环岛点位
- T 型路口
- 十字路口
- Y 型路口

### 4.2 设备安装位置

设备安装位置是设备落位管理的最小业务单元。一个点位下可有多个安装位置，如：

- 东
- 西
- 南
- 北
- 路口中心
- 机箱
- 杆件
- 交警支队机房

设备安装位置与点位通过 `NodeID` 关联。

### 4.3 一点一档

一点一档是点位完整档案，应汇总：

- 点位基础信息
- 高德 GCJ-02 展示坐标
- 原始 CGCS2000 坐标审计记录
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 可变车道信息
- 设备安装清单
- 合同与供应商信息
- 施工、验收、入库、领料单据
- 照片、图纸、报告、工单和运维记录

## 5. 点位管理功能修订

### 5.1 点位三视图

点位管理模块必须提供三种视图，可在同一页面内通过视图切换控件切换。

#### 表格视图

适合批量筛选、导入导出、字段维护和数据核对。

核心能力：

- 点位列表不分页，内容区通过右侧滚动条浏览
- 多条件筛选
- 列显示配置
- 支持拖动调整列顺序和列宽
- 有异常点位默认排在表格前面
- 行政区域用不同颜色标签显示
- 批量导入
- 批量导出
- 批量修改行政区域、点位状态、信号机厂商等字段
- 点击行进入点位编辑页

#### 卡片视图

适合项目管理人员快速浏览点位状态与关键摘要。

每张卡片展示：

- 点位名称
- NodeID
- 点位类型
- 行政区域
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 是否可变车道路口
- 设备数量
- 安装完成数量
- 当前状态

点击卡片任意区域进入点位编辑页。

#### 地图视图

适合从空间维度查看点位分布与建设态势。

地图视图使用高德 Web JS API/地图 SDK。系统应将点位表中的 CGCS2000 坐标转换为 GCJ-02 坐标后再渲染到高德地图。

地图能力：

- 点位聚合
- 按点位类型、行政区域筛选
- 不同行政区域点位以不同颜色标识
- 按 NodeID、点位名称搜索
- 点击地图点位弹出摘要卡片
- 点击摘要卡片进入点位编辑页
- 支持当前视野内点位列表联动

### 5.2 点位编辑页面

点击表格行、卡片或地图点位后进入点位编辑页面。

页面建议采用 `左侧基础表单 + 右侧嵌入地图 + 下方页签` 的布局。

#### 顶部标题区

展示：

- 点位名称
- NodeID
- 行政区域
- 点位类型
- 当前状态
- 保存、取消、提交审核、查看一点一档、创建工单等操作按钮

#### 基础信息区

维护点位基础字段：

- 序号
- 点位类型
- 行政区域
- NodeID
- CrossID
- 点位英文名称（NodeName）
- 点位名称
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 是否可变车道路口
- 可变车道数量
- 可变车道方向

#### 嵌入式地图区

地图必须满足：

- 默认中心为当前点位。
- 默认缩放级别需能清晰显示当前点位和周边 4-5 个最近点位。
- 当前点位用高亮样式展示。
- 周边点位用弱化样式展示，并显示简短标签。
- 点击周边点位可打开该点位摘要，可选择跳转到对应点位编辑页。
- 地图上所有点位均使用 GCJ-02 坐标展示；原始 CGCS2000 坐标仅通过审计入口查看。

周边点位计算规则：

1. 以当前点位 GCJ-02 坐标为中心。
2. 计算所有同项目点位与当前点位的球面距离。
3. 排除当前点位本身。
4. 取最近 4-5 个点位。
5. 若附近点位不足 4 个，则展示当前行政区域内最近点位补足。

#### 设备信息页签

展示该点位关联设备安装位置表中的设备信息。

列表默认按设备类型逐个显示设备，并可按安装位置、供应商、物料名称分组，字段包括：

- 安装位置
- 供应商单位
- 物料编码
- 物料名称
- 合同名称
- 品牌型号
- 单位
- 合同数量
- 已送货数量
- 入库数量
- 开箱验收数量
- 本期领料数量
- 已安装数量
- 在仓库数量
- 相关单据索引
- 单据日期
- 开箱验收问题

#### 一点一档页签

一点一档页签以档案形式汇总当前点位的全部信息。

建议分组：

- 基础档案：NodeID、CrossID、点位名称、行政区域、点位类型、信号机厂商。
- 坐标与地图：GCJ-02 坐标、当前点位地图、周边点位距离；原始 CGCS2000 仅用于审计记录。
- 信号系统信息：信号机厂商、信号系统资料、相关附件。
- 感知与自适应信息：感知点位类型、是否自适应点位、相关设备摘要。
- 可变车道信息：是否可变车道路口、可变车道数量、可变车道方向。
- 设备安装清单：按设备类型逐项展示安装位置、供应商、物料、合同、数量和状态。
- 合同与供应商：后项合同、供应商、合同板块、合同金额和关联合同流。
- 送货/入库/领料/开箱验收单据：单据编号、日期、数量、附件索引和问题。
- 施工照片与验收资料：施工照片、验收记录、一路一档资料。
- 运维工单：问题描述、处理状态、闭环结果。
- 操作日志：导入、编辑、匹配、删除等审计记录。

## 6. 设备管理功能修订

### 6.1 设备数据来源

设备管理一期以 `设备安装位置表.xlsx` 为基础数据源。`Sheet1` 中有 NodeID 的数据进入点位设备清单；`未匹配` 工作表进入待治理设备池。

### 6.2 设备台账粒度

设备台账建议分为两层，并在点位详情中按设备类型逐个展示：

1. 设备安装汇总项：来自 Excel 的每一行安装位置记录。
2. 设备资产实例：当存在 SN、二维码或实物资产编号时，拆分成单台设备实例。

若 Excel 中只有数量，没有 SN，则先按安装汇总项管理；后续入库扫码或序列号验证后再拆分为资产实例。

### 6.3 设备与点位关联

关联规则：

- 主关联键：`NodeID`
- 辅助匹配：`点位名称`
- 设备落位：`安装位置`
- 设备识别：`物料编码 + 物料名称 + 品牌型号`

若设备记录无 NodeID，则进入未匹配池，支持人工选择：

- 关联到点位
- 关联到机房
- 关联到仓库
- 标记为车端设备
- 标记为软件/服务类合同项
- 标记为不纳入点位设备统计

### 6.4 设备状态

设备状态由数量字段和单据字段综合判断：

- 合同内：存在后项合同数量
- 已送货：已送货数量大于 0
- 已入库：入库单数量大于 0
- 已开箱验收：开箱验收单数量大于 0
- 已领料：本期领料数量大于 0
- 已安装：已安装数量大于 0
- 在仓库：在仓库数量大于 0
- 验收异常：开箱验收问题非空

### 6.5 运维管理离线/异常设备点位地图

运维管理功能页应提供“离线/异常设备点位地图”，用于展示每日路侧设备运行状态快照中的离线、异常设备分布。该地图不按设备自身经纬度打点，而按设备所属点位打点。

点位关联与落图规则：

- 从路侧设备运行状态表读取 `路口ID` 字段。
- 用 `路口ID` 匹配点位管理表中的 `NodeId` 字段。
- 地图 marker 坐标使用匹配到的点位管理表坐标，地图展示仍统一使用 GCJ-02 坐标。
- 同一 `NodeId` 下存在多台离线或异常设备时，地图上只显示一个点位 marker，不按设备分散显示。
- 未匹配到点位管理表的离线、异常设备不落图，应在地图区域提示未匹配数量，并进入数据治理清单。

点位信息框展示规则：

- 鼠标移动到离线/异常点位 marker 上方时，显示该点位的信息框。
- 信息框顶部显示点位名称和 NodeID。
- 信息框中按设备类型聚合展示：设备类型、设备数量、离线数量、异常数量。
- 不使用表格控件展示上述信息，采用普通信息行或分组行展示。
- 离线数量和异常数量必须使用红色字体表示，包括对应数字。
- 高德地图 InfoWindow 与静态回退 tooltip 的展示内容和统计口径应保持一致。
- 鼠标悬停打开信息框时，地图不得因信息框弹出而自动平移或抖动；前端应关闭 InfoWindow 自动平移行为。

### 6.6 运维管理异常日历与离线/异常趋势

运维管理功能页应提供异常日历，用于查看不同日期的路侧设备运行状态快照。用户点击异常日历中的某一天后，页面应切换到该日期的统计口径，并同步刷新离线/异常设备点位地图、离线/异常趋势图和按设备类型统计信息。

异常日历展示规则：

- 日历数据来源为已导入并保存的路侧设备运行状态历史快照。
- 每个有数据的日期应展示当日离线数量、异常数量和异常占比。
- 无历史快照的日期不得误显示为有数据状态，可保留导入入口或空状态提示。
- 当日状态表导入后，原当前状态表应进入历史档案库，新文件成为当前路侧设备状态表。

离线/异常趋势图展示规则：

- “离线/异常趋势”图表必须作为独立卡片展示，不嵌套在异常日历卡片或地图卡片内部。
- 该趋势卡片应布置在异常日历卡片和离线/异常设备点位地图卡片下方，宽度自动铺满工作页面。
- 点击异常日历某一日期后，趋势图展示从已有历史数据起始日期到选中日期的离线数量和异常数量变化。
- 趋势图应使用平滑折线展示离线数量和异常数量；离线和异常应使用不同颜色区分。
- X 轴必须显示对应日期。日期较多时允许进行合理密度控制，但必须保留可识别的日期刻度。
- 图表背景网格应同时包含横向数值线和竖向日期线，形成网格状背景。
- 鼠标移动到离线或异常趋势线上方时，应显示对应日期的离线数量和异常数量。
- 趋势图卡片在不同视觉风格下应继承同类卡片的材质、圆角、阴影和透明度规则，不得单独使用与其他卡片冲突的卡片外观。

### 6.7 运维状态数据持久化与统计口径

路侧设备运行状态历史数据应支持本地持久化，避免浏览器刷新或重新打开页面后丢失导入历史。原型阶段可通过本地 Node.js 服务将运维状态快照保存到工作目录内的本地文件；生产阶段应落库保存。

统计口径：

- 仅启用状态的路侧设备纳入离线、异常和按设备类型统计。
- “离线数量”按状态字段为离线的设备计数。
- “异常数量”按运行状态中判定为异常的设备计数；离线设备是否同时纳入异常，应在后端口径中明确并保持全页面一致。
- 离线/异常设备点位地图按点位聚合，趋势图按日期聚合，按设备类型统计按设备类型聚合。
- 所有运维统计模块应使用同一份日期快照数据，避免地图、日历、趋势图和表格之间出现口径差异。

### 6.8 应用视觉风格切换

系统应提供视觉风格切换能力，用于不同演示和运维场景。视觉风格切换不应改变业务数据、筛选条件、日期选择和导入状态。

轨迹风格视觉规范：

- 页面背景为纯哑光黑，不使用反光背景效果。
- 卡片采用磨砂半透明玻璃面板，透明度口径为 35%，边缘做厚实圆润处理，类似围棋子般的圆润边缘。
- 卡片边缘应有微倒角高光和柔和投影，但同一风格下所有卡片材质必须保持一致。
- 卡片文字不得使用“上高光、下暗影”的 text-shadow 压印方式；应使用与卡片底色对比更强的橙色、蓝色、红色或高亮白色。
- 运维管理页在轨迹风格下主标题显示为“轨迹实时指标”。
- 数据口径、异常日历、离线/异常设备点位地图、离线/异常趋势和统计表卡片不得覆盖功能菜单，不得出现漂浮到侧边菜单上方的布局问题。
- 图表在轨迹风格下应呈现立体玻璃质感，但不得破坏卡片统一风格和数据可读性。

## 7. 坐标与地图规则

### 7.1 坐标存储

系统必须同时保存：

- `longitude_cgcs2000`：原始规划经度
- `latitude_cgcs2000`：原始规划纬度
- `longitude_gcj02`：高德地图展示经度
- `latitude_gcj02`：高德地图展示纬度
- `coordinate_convert_status`：坐标转换状态
- `coordinate_convert_time`：转换时间

### 7.2 坐标转换

Excel 中经纬度标注为 CGCS2000。高德地图使用 GCJ-02。系统导入或保存点位时，应执行 CGCS2000 到 GCJ-02 转换。

产品口径：

- 原始坐标不可覆盖。
- 转换坐标用于地图展示、距离计算和周边点位查询。
- 若转换失败，点位仍可保存，但地图展示为异常状态。
- 支持手工重新转换。

技术口径：

- 无锡范围内 CGCS2000 地理坐标与 WGS84 地理坐标差异较小，工程实现可按 `CGCS2000 ≈ WGS84` 输入火星坐标转换算法得到 GCJ-02。
- 如业主提供精确七参数或测绘转换服务，应替换为权威转换服务。
- 所有转换结果需保留转换方式与时间，方便审计。

### 7.3 高德 Web JS API/地图 SDK

地图能力通过高德 Web JS API/地图 SDK 接入，前端只消费系统后端返回的 GCJ-02 坐标。

一期地图场景：

- 点位管理地图视图
- 点位编辑页嵌入式地图
- 周边点位展示
- 点位搜索定位
- 点位状态图层

## 8. 数据治理要求

### 8.1 点位导入治理

导入点位管理表时必须校验：

- NodeID 不为空且唯一。
- 经纬度字段可解析为数字。
- 行政区域必须落在系统字典内。
- 点位类型必须落在系统字典内。
- 重复 NodeID 禁止导入。
- 坐标缺失的点位进入坐标异常清单。

### 8.2 设备导入治理

导入设备安装位置表时必须校验：

- Sheet1 中 NodeID 必须能匹配点位库。
- 未匹配工作表进入设备未匹配池。
- 同一 NodeID 下允许多条安装位置记录。
- 数量字段必须为数字或空值。
- 单据索引字段作为业务追溯凭证保存。
- 开箱验收问题非空时生成设备验收异常。

### 8.3 未匹配数据治理

未匹配设备池支持：

- 展示 `未匹配` 工作表中的 2447 条记录。
- 按物料、供应商、合同板块、合同名称、安装位置筛选。
- 人工补充 NodeID。
- 批量标记资产类型。
- 批量关联机房/仓库。
- 支持治理到点位、仓库、机房或服务项。
- 批量排除点位统计。
- 治理完成后进入正式设备台账。

## 9. 合同管理功能补充

无锡车路云一体化项目合同结构复杂，总集单位为移动+天安。系统合同管理不能只做合同台账，应支持合同流图谱、分项包、设备级合同明细和点位设备穿透。

### 9.1 合同流管理

系统应导入 `无锡车路云一体化采购价格测算表202500920.xlsx` 中的 `合同流` sheet，自动拆解：

- 分项编号
- 项目名称
- 投标价
- 拆分包名称
- 拆分包价格
- 一级至六级分包商
- 一级至六级分包价格
- 原厂价格
- 毛利
- 折扣率
- 供应商
- 对应明细 sheet

合同流需要支持以下链路：

- 移动向后流动。
- 移动流回天安后由天安继续向后流动。
- 天安直接向后流动。
- 移动自有或天安自有。
- 暂不实施、服务类、施工类、资源类条目。

### 9.2 设备级合同管理

系统应导入各分项明细 sheet，将硬件、软件、服务、施工等明细统一形成合同明细台账。

硬件类明细需要管理到设备级，并与 `设备安装位置表.xlsx` 关联：

```text
合同明细
→ 物料名称/品牌型号/合同链路
→ 设备安装项
→ NodeID
→ 点位
→ 到货/入库/领料/安装/验收
```

注意：同一个分项 sheet 中的不同设备可能分属不同前向合同主体、后向合同主体和原厂供应商，因此合同归属必须下沉到设备/明细行级，不能按 sheet 统一继承。

### 9.2.1 前向销售与后向采购合同

合同管理需要扫描并管理：

- 前项天安销售合同目录：天安作为销售/乙方，对接浪潮、上汽帆一、中通服网盈、工业安装、中城工联、合创智行、四维图新等。
- 后项天安采购盖章合同目录：天安作为采购/甲方，对接万集、航天海特、华通、海康智联、中电鸿信、四维图新、隆顺、车城智联、联通华盛、工业安装等。

系统需支持合同 PDF OCR 解析和人工复核，提取合同编号、甲方、乙方、合同金额、签署日期、付款条款、交付验收条款，并与合同流、分项明细、设备安装项关联。

主体归一规则：

- `尚行`、`上海帆一尚行科技有限公司`、`帆一` 统一显示为 `上汽帆一`。

### 9.2.2 设备级履约与资金流

同一合同内，不同设备可能分批到货、入库、安装、验收和付款。系统需要在设备级记录每个节点的发生时间，并将资金流细化到设备/合同明细级。

设备级事件包括：

- 下单
- 送货
- 到货查验
- 入库
- 开箱验收
- 领料
- 安装
- 初验
- 试运行
- 终验
- 审计确认
- 质保开始/结束

资金流包括：

- 前向应收/实收
- 后向应付/实付
- 质保金
- 回流业主车联网集团

后向采购合同中如存在背靠背付款条款，后向付款节点必须绑定上游回款状态。若上游未回款，后向付款节点显示为“待上游回款”。

天安自有或集成费不能简单作为自有项关闭，其中部分需要通过采购数据、设备清单、服务确认或审计结果回流到业主车联网集团，系统必须标记并跟踪回流状态。

### 9.3 点位设备合同穿透

点位编辑页的设备信息页签应增加合同穿透字段：

- 合同流路径
- 分项编号
- 分项名称
- 合同明细名称
- 上游单位
- 下游单位
- 供应商
- 合同数量
- 合同单价
- 合同金额
- 原厂单价
- 原厂采购金额
- 毛利
- 合同签署状态
- 履约状态

### 9.4 合同风险

系统应自动识别：

- 合同链路断裂。
- 投标价、拆分包价格、各级流转价、原厂价不一致。
- 合同硬件明细未匹配设备。
- 设备安装项未匹配合同。
- 合同数量、入库数量、领料数量、安装数量不一致。
- 合同签署计划逾期。
- 负毛利或价格倒挂。

## 10. 一期验收补充标准

点位管理验收：

- 点位表格视图、卡片视图、地图视图均可正常切换。
- 任一点位均可从三种视图进入点位编辑页。
- 点位编辑页能展示嵌入式地图。
- 嵌入式地图能高亮当前点位并显示周边 4-5 个点位。
- 点位编辑页能展示该点位设备清单。
- 点位编辑页能展示一点一档信息。
- 业务页面显示点位坐标全部以 GCJ-02 坐标为准。
- 原始 CGCS2000 坐标仅用于原始数据记录和审计。

设备管理验收：

- 能导入设备安装位置表 Sheet1。
- 能按 NodeID 关联设备到点位。
- 能展示每个点位下的设备安装位置和物料清单。
- 能识别并展示开箱验收问题。
- 能导入未匹配工作表并进入待治理池。
- 未匹配设备支持人工治理并关联到点位或非点位资产。
