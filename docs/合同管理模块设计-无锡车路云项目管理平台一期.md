# 合同管理模块设计：无锡车路云项目管理平台一期

## 1. 设计结论

本模块已推翻旧的旧静态合同表式、静态图式式、sheet 明细台账式设计。新版合同管理模块定义为：

```text
合同关系重构与设备级阶段资金流管理模块
```

核心目标不是“展示有哪些合同”，而是回答：

```text
每一笔前向合同收入经过天安后，拆给了哪些后向合同、哪些设备/软件/服务、哪些点位、哪些审计批次，以及哪些付款阶段可以释放。
```

## 2. 项目合同关系背景

无锡车路云一体化项目甲方为无锡市车联网产业集团，乙方为移动与天安组成的联合体。移动和天安承担的合同执行工作量金额大致为 9:1，但合同流不是简单一层分包：

- 移动侧合同流大量经过天安回流。
- 天安收到前向销售合同后，会拆解为多个后向采购合同。
- 多个前向合同也可能各取部分设备、服务或金额，汇入一个或多个后向采购合同。
- 天安是技术主导方和兜底方。
- 上游回款需要经过业主设备级跟踪审计。
- 下游付款需要根据设备级审计结果和上游回款结果按阶段释放。

## 3. 关键建模原则

### 3.1 Excel sheet 不是合同边界

必须严格遵守：

- Excel 各分项 sheet 是设备、软件、服务清单池，不是合同。
- 一个 sheet 可能对应一个合同、多个合同、部分合同。
- 多个 sheet 也可能共同对应一个合同。
- 合同边界必须来自正式前向销售合同和后向采购合同文本。
- Excel `合同流` sheet 尾部才是宏观资金流骨架。

禁止再使用：

```text
sheet → 合同流
```

作为合同关系确认逻辑。

### 3.2 前后向合同是多对多关系

正确关系是：

```text
前向合同中的某部分设备/服务/金额
→ 经天安
→ 对应到一个或多个后向合同中的某部分设备/服务/金额
```

### 3.3 资金流必须钻取到设备级和阶段级

不能只记录：

```text
某合同已回款多少 / 已付款多少
```

必须记录：

```text
设备 / 合同明细 / 点位 / 审计批次 / 付款阶段 / 审计确认金额 / 上游回款 / 下游付款释放
```

## 5. 宏观合同流修正规则

必须使用：

```text
移动-四维图新-天安（大华+海康智联+RSU)
```

禁止使用：

```text
旧的“吉利/希迪版本”表述
```

说明：

- 该流按四维图新理解。
- 后向括号内为大华、海康智联和 RSU 相关设备。
- `RSU` 是设备，不是合同主体。
- 主体链可解析为：

```text
移动 → 四维图新 → 天安 → 海康智联 → 大为
```

## 6. 页面设计

合同管理页面是“合同关系重构工作台”，包含五个区域。

### 6.1 宏观合同流骨架

展示 Excel `合同流` 尾部识别出的宏观流，例如：

```text
车联网-天安（华通+交科所+隆顺）
移动-浪潮-天安（大为信号机+R1）
移动-上研-合创-天安（应用平台）
移动-上研-工业安装-天安（华通+公交硬件+R3等）
移动-中通服网盈-天安(R2)
移动-尚行-天安(车城+服务器）
移动-四维图新-天安（大华+海康智联+RSU)
```

用户点击某条宏观合同流后，系统筛选相关 Word 合同和前后向候选关系。

### 6.2 Word 合同匹配概览

展示：

- 前向/后向。
- 合同名称。
- 主体链。
- 税率。
- 可能匹配的宏观合同流。
- 匹配依据。

### 6.3 前向合同 → 天安 → 后向合同多对多候选关系

每行代表一个候选关系：

```text
某前向合同可能通过天安对应到某后向合同
```

展示：

- 置信度。
- 前向合同。
- 后向合同。
- 共同宏观合同流。
- 税率重合。
- 主体/关键词/金额/付款条款证据。
- 状态：待确认、已确认、已否决。

候选关系不得进入资金计算。必须人工确认。

### 6.4 设备级资金流单元

确认前后向合同关系后，继续拆到设备级：

```text
前向合同明细
→ 后向合同明细
→ Excel 清单池行
→ NodeID/点位/安装位置
→ 业主审计
→ 上游回款
→ 后向付款释放
```

### 6.5 阶段付款释放看板

设备级资金流仍需按阶段部分释放：

```text
预付款 / 到货款 / 安装款 / 验收款 / 审计款 / 质保金
```

## 10. 付款释放规则

```text
预付款：
  合同签署和预付款条款满足；
  背靠背条款下需等待上游预付款到账。

到货款：
  根据到货、入库、发票和对应上游回款释放。

安装款：
  根据安装签证、安装数量和对应上游回款释放。

验收款：
  根据验收单、验收数量和对应上游回款释放。

审计款：
  根据业主审计确认数量/金额释放；
  付款基数必须以审计确认金额为准。

质保金：
  根据质保期、遗留问题和最终审计确认金额释放。
```

前向合同和后向合同阶段可能不一致。系统应支持：

```text
前向设备级阶段回款
→ 设备级资金池
→ 后向设备级阶段付款释放
```

## 11. 风险识别

系统应识别：

- 前向合同已确认但未拆到设备。
- 后向合同找不到对应前向资金来源。
- 同一设备被多个后向合同重复认领。
- 前向已审计但未回款。
- 后向申请付款但对应设备未审计。
- 审计确认金额小于合同金额。
- 前后向税率不一致。
- 前向金额不足覆盖后向付款。
- 已回款金额无法追溯到具体设备。
- 后向阶段付款超过可释放金额。
- 付款比例被误识别为税率。
