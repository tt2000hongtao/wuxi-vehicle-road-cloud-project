# 页面原型说明书：无锡车路云项目管理平台一期

## 1. 原型范围

本文档描述一期重点页面的信息架构、布局、交互和校验规则。重点覆盖：

- 点位管理三视图
- 点位编辑页面
- 点位编辑页嵌入式地图
- 一点一档信息
- 设备管理与设备安装位置
- 设备未匹配治理
- 合同管理与点位设备穿透

### 1.1 项目总览页面

总览卡片入口：

- 项目点位：点击进入点位管理功能页。
- 设备记录：点击进入设备管理功能页。
- 未匹配设备：点击进入未匹配池功能页，展示 2447 条未匹配记录，支持按物料、供应商、合同板块治理到点位、仓库、机房或服务项。
- 合同流：点击进入合同管理功能页；卡片内容分为天安前向合同和天安后向合同两个数据，卡片名称不使用合同流明细行口径。

总览页顶部操作：

- 导入点位管理表：用于导入点位管理表并生成/更新点位库，模板格式参考 `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/点位管理表-无锡车路云.xlsx`。
- 导入设备管理表：用于导入设备点位分布信息并按 NodeID 关联点位，模板格式参考 `/Users/tt2000/Documents/天安智联/AI/项目管理工具包/Project Data/设备安装位置表.xlsx`。

## 2. 点位管理页面

### 2.1 页面目标

点位管理页面用于项目人员以表格、卡片、地图三种方式查看和管理点位。三种视图共享同一套筛选条件。

### 2.2 顶部筛选区

筛选项：

- NodeID
- 点位名称
- 点位英文名称
- 点位类型
- 行政区域
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 是否可变车道路口

操作：

- 查询
- 重置
- 导入点位表
- 导出当前结果
- 列配置
- 视图切换：表格 / 卡片 / 地图

### 2.3 表格视图

表格列：

- 序号
- 点位类型
- 行政区域
- NodeID
- CrossID
- 点位英文名称
- 点位名称
- 高德展示经度 GCJ-02
- 高德展示纬度 GCJ-02
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 是否可变车道路口
- 可变车道数量
- 可变车道方向
- 设备记录数
- 已安装数量
- 异常数量
- 更新时间

行操作：

- 编辑
- 查看一点一档
- 查看地图
- 查看设备
- 创建工单
- 删除

交互规则：

- 不分页显示，表格内容区通过右侧滚动条浏览。
- 默认将有异常点位排在表格前面。
- 点击整行或点位信息进入点位编辑页面。
- 点击 `删除` 时必须二次确认；若点位下挂有设备，需先到设备管理删除该点位下设备后，才能删除点位。
- 支持列配置、拖动调整列顺序、拖动调整列宽。
- 行政区域以不同颜色标签显示。
- `NodeID` 缺失或重复的导入记录不得进入正式点位库。
- 设备记录数为 0 的点位显示弱提示，便于核对设备表匹配情况。

- 地图搜索支持 NodeID 和点位名称。

## 3. 点位编辑页面

### 3.1 页面目标

点位编辑页面用于维护单个点位的完整档案，并展示当前点位地图、周边点位、设备安装信息和一点一档资料。

### 3.2 页面结构

```text
------------------------------------------------------+
| 标题区：点位名称 / NodeID / 状态 / 操作按钮          |
+-----------------------------+------------------------+
| 基础信息表单                | 嵌入式地图             |
|                             | 当前点位 + 周边点位    |
+-----------------------------+------------------------+
| 页签：设备信息 / 一点一档 / 单据资料 / 工单 / 日志   |
+------------------------------------------------------+
```

### 3.3 标题区

展示：

- 点位名称
- NodeID
- 行政区域
- 点位类型

操作按钮：

- 保存
- 取消
- 重新转换坐标
- 查看一点一档
- 创建工单
- 返回列表

### 3.4 基础信息表单

字段：

- 序号
- 点位类型
- 行政区域
- NodeID
- CrossID
- 点位英文名称（NodeName）
- 点位名称
- 高德展示经度 GCJ-02
- 高德展示纬度 GCJ-02
- 信号机厂商
- 感知点位类型
- 是否自适应点位
- 是否可变车道路口
- 可变车道数量
- 可变车道方向

校验规则：

- NodeID 必填且唯一。
- 点位名称必填。
- 行政区域必选。
- 点位类型必选。
- 原始经纬度必须为合法数字。
- 原始 CGCS2000 坐标仅用于原始数据记录和审计；业务页面默认只显示 GCJ-02 坐标。

### 3.5 嵌入式地图

展示要求：

- 使用高德 Web JS API/地图 SDK。
- 地图中心为当前点位 GCJ-02 坐标。
- 当前点位高亮显示并展示标签。
- 周边 4-5 个最近点位以普通点位样式展示。
- 自动缩放到同时包含当前点位和周边点位的范围。

地图信息面板：

- 当前点位 GCJ-02 坐标
- 转换方式
- 转换时间
- 原始 CGCS2000 坐标审计入口
- 周边点位列表与距离

周边点位列表字段：

- 距离排名
- NodeID
- 点位名称
- 行政区域
- 点位类型
- 距离米

交互规则：

- 点击周边点位，在地图上打开摘要气泡。
- 摘要气泡提供“跳转编辑页”。
- 若当前点位坐标转换失败，地图区域显示异常提示，并提供重新转换按钮。

## 5. 点位编辑页：一点一档页签

### 5.1 页面目标

以档案形式汇总当前点位的完整资料，支持查看、导出和打印。

### 5.2 档案结构

档案分区：

1. 基础档案：NodeID、CrossID、点位名称、点位英文名称、行政区域、点位类型、信号机厂商。
2. 坐标与地图：GCJ-02 坐标、当前点位高亮地图、周边点位距离列表；原始 CGCS2000 坐标仅在审计记录中查看。
3. 信号系统信息：信号机厂商、信号系统资料、相关附件。
4. 感知与自适应信息：感知点位类型、是否自适应点位、相关感知设备摘要。
5. 可变车道信息：是否可变车道路口、可变车道数量、可变车道方向。
6. 设备安装清单：按设备类型逐项列出安装位置、供应商、物料、合同、数量、状态和问题。
7. 合同与供应商：后项合同、供应商、合同板块、合同金额和关联合同流。
8. 送货/入库/领料/开箱验收单据：单据编号、日期、数量、附件索引、开箱验收问题。
9. 施工与验收资料：施工照片、验收记录、一路一档资料。
10. 运维工单：问题描述、处理状态、闭环结果。
11. 操作日志：导入、编辑、匹配、删除等审计记录。

操作：

- 导出 PDF
- 导出 Word
- 打印
- 上传资料
- 补充备注

## 9. 合同管理页面

新版合同管理页面不再是静态旧静态合同表，也不再让用户按 Excel 清单工作表确认合同边界。页面定位为：

```text
合同关系重构工作台
+ 宏观合同流骨架
+ Word 合同解析复核
+ 前向合同 → 天安 → 后向合同多对多候选关系确认
+ 设备级资金流单元
+ 业主设备级审计跟踪
+ 阶段付款释放看板
```

### 9.1 合同关系重构工作台

页面顶部展示当前重构状态，而不是展示静态金额卡片：

- 宏观合同流数量。
- Word 合同数量。
- 前向合同数量。
- 后向合同数量。
- 前后向候选关系数量。
- 已确认、待确认、已否决数量。
- 设备级资金流单元生成状态。
- 高风险锁定项数量。

原型状态说明：确认结果暂存 `localStorage`，正式系统应写入后端审核表。

### 9.2 宏观合同流骨架

展示 Excel `合同流` 工作表尾部识别出的宏观资金骨架。该骨架用于理解资金链路，不用于直接生成合同边界。

交互要求：

- 默认选中第一条宏观合同流。
- 点击某条宏观合同流后，筛选相关 Word 合同和前后向候选关系。
- 可切回“全部宏观流”。
- 每条宏观流展示：Excel 行号、标准合同流名称、主体链路、天安侧金额、链路类型。

必须使用修正名称：

```text
移动-四维图新-天安（大华+海康智联+RSU)
```

其中 `RSU` 是设备，不是合同主体。

### 9.3 Word 合同解析复核

Word 合同文本决定合同边界。列表展示每份合同的程序化解析结果：

- 合同方向：前向销售 / 后向采购。
- 合同文件名。
- 主体链。
- 合同金额线索。
- 税率线索。
- 设备/服务关键词。
- 匹配到的宏观合同流。
- 解析可信度和待复核状态。

用户需要在正式系统中确认：方向、主体、金额、税率、付款条款和归属宏观流是否正确。

### 9.4 前向合同 → 天安 → 后向合同候选关系

这是页面核心工作区。每行表示一个候选关系：

```text
某前向销售合同
→ 经天安
→ 某后向采购合同
```

候选行必须展示：

- 置信度。
- 前向合同名称。
- 后向合同名称。
- 共同宏观合同流数量。
- 税率重合情况。
- 主体、设备/服务关键词、付款条款等证据。
- 状态：待确认、已确认、已否决。

交互要求：

- 按宏观合同流筛选。
- 按确认状态筛选。
- 按置信度筛选。
- 按税率筛选。
- 按合同名、主体或设备关键词搜索。
- 查看证据明细。
- 对候选关系执行“确认”“否决”“恢复为候选”。

候选关系未确认前，绝不能进入应收、应付、毛利或付款释放计算。

### 9.5 设备级资金流单元

合同级关系确认后，继续拆到设备/软件/服务项和点位级。最小管理单元为：

```text
前向合同明细
→ 后向合同明细
→ 清单池行
→ NodeID / 点位 / 安装位置
→ 业主设备级审计
→ 上游阶段回款
→ 后向阶段付款释放
```

页面应展示：

- 设备/服务名称。
- 规格型号。
- NodeID 和点位。
- 前向合同、金额、税率。
- 后向合同、金额、税率。
- 数量、价差、毛利。
- 审计状态。
- 当前回款阶段。
- 当前付款阶段。
- 锁定原因和风险等级。

### 9.6 业主设备级审计跟踪

上游回款不是合同整体回款判断，而是设备级审计结果驱动。

必须支持状态：

- 待审计。
- 已提交。
- 审计通过。
- 部分通过。
- 审计不通过。
- 需补资料。

只有通过业主设备级审计并满足合同条款的设备，才可以进入对应阶段的上游回款判断。

### 9.7 阶段付款释放看板

设备级资金流仍需按阶段部分释放，至少包含：

```text
预付款 / 到货款 / 安装款 / 验收款 / 审计款 / 质保金
```

每个阶段展示：

- 触发条件。
- 业主审计要求。
- 上游回款要求。
- 后向付款释放规则。
- 是否允许部分释放。
- 是否被审计结果锁定。
- 是否存在上游未回款但下游付款风险。

### 9.8 点位/设备穿透

合同管理页面应能向下钻取到：

- 点位。
- NodeID。
- 设备安装记录。
- 验收资料。
- 审计结果。
- 出入库单据。
- 合同原文和附件。
- 付款记录。

点位页和设备页展示的合同信息，应来自已确认的设备级资金流单元，而不是来自清单工作表名称。
