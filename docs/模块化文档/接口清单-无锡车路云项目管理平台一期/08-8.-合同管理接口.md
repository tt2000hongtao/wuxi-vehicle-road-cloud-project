### 8.2 导入合同测算表预校验

```text
POST /api/v1/import/contracts/preview
```

请求：

- multipart file：`无锡车路云一体化采购价格测算表202500920.xlsx`

处理内容：

- 读取 `合同流`
- 读取 `合同签署计划`
- 读取 `分项报价汇总表`
- 读取各分项明细 sheet

返回字段：

- importTaskId
- flowLineCount
- packageCount
- contractItemCount
- partyCount
- detailSheetCount
- parseWarningCount
- previewFlowLines

### 8.3 确认导入合同测算表

- importedSignPlans

### 8.3.1 扫描前向/后向合同文件目录

```text
POST /api/v1/import/contracts/documents/scan
```

用途：扫描天安前项销售、后项采购盖章合同目录，生成合同文件索引，并尝试 OCR 识别合同编号、甲乙方、金额、签署日期和合同清单。

请求体：

```json
{
  "contractDirection": "front_sales",
  "rootPath": "/Users/tt2000/Documents/天安智联/0000无锡市车路云一体化项目/0.合同/合同管理/前项天安销售(1)"
}
```

`contractDirection` 可选：

- `front_sales`：天安前向销售合同
- `back_purchase`：天安后向采购合同

返回字段：

- scannedFiles
- parsedFiles
- parseFailedFiles
- createdContracts
- suspectedContracts
- ocrWarningCount

### 8.3.2 合同主体归一

```text
POST /api/v1/contract-parties/normalize
```

用途：执行合同主体名称归一，例如尚行、上海帆一尚行、帆一归一为上汽帆一。

请求体：

```json
{
  "rules": [
    {
      "standardPartyName": "上汽帆一",
      "aliases": ["尚行", "上海帆一尚行科技有限公司", "帆一"]
    }
  ]
}
```

### 8.17 生成设备级资金流计划

```text
POST /api/v1/contracts/{contractId}/cash-flows/generate
```

用途：根据合同付款节点、合同明细、设备匹配和履约事件，生成设备级应收/应付/质保金/业主回流计划。

请求体：

```json
{
  "allocationMethod": "by_quantity",
  "respectUpstreamReceipt": true,
  "includeOwnerBackflow": true
}
```
