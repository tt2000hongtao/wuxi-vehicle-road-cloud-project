## 8. 合同管理接口

合同管理接口按“合同关系重构工作台”设计。接口返回候选关系和证据，人工确认后才允许生成设备级资金流单元；未确认候选关系的 `financialUseAllowed` 必须为 `false`。

### 8.1 合同关系重构工作台概览

```text
GET /api/v1/contract-relationship-workbench/summary
```

返回字段：

- macroFlowCount
- contractDocumentCount
- frontContractCount
- backContractCount
- frontBackCandidateCount
- pendingCandidateCount
- confirmedCandidateCount
- rejectedCandidateCount
- deviceCashflowUnitCount
- ownerAuditPendingCount
- paymentStageLockedCount
- taxRateCounts

### 8.2 导入合同测算表预校验

```text
POST /api/v1/import/contracts/workbook/preview
```

处理内容：

- 读取 `合同流` sheet 尾部宏观资金骨架。
- 读取各清单池工作表作为设备/软件/服务明细血缘。
- 不把任意清单池工作表当成合同边界。

返回字段：

- importTaskId
- macroFlowCount
- itemPoolSheetCount
- itemPoolRowCount
- parseWarningCount
- previewMacroFlows

### 8.3 确认导入合同测算表

```text
POST /api/v1/import/contracts/workbook/confirm
```

请求体：

```json
{
  "importTaskId": "xxx"
}
```

返回字段：

- importedMacroFlows
- importedItemPoolRows
- importedPartyAliases
- warningCount

### 8.4 扫描 Word 前后向合同文件目录

```text
POST /api/v1/import/contracts/documents/scan
```

用途：扫描 Word 前向销售、后向采购合同目录，生成合同文本边界索引，并尝试解析合同编号、甲乙方、金额、税率、付款条款、验收条款和背靠背付款线索。

请求体：

```json
{
  "rootPath": "/Users/tt2000/Documents/天安智联/0000无锡市车路云一体化项目/0.合同/盖章版合同 WORD 版"
}
```

返回字段：

- scannedFiles
- parsedFiles
- parseFailedFiles
- frontContractCount
- backContractCount
- taxRateCounts
- createdContractDocuments
- pendingReviewCount

### 8.5 合同主体归一

```text
POST /api/v1/contract-parties/normalize
```

用途：执行合同主体名称归一。注意 `RSU`、`OBU`、`GPU`、`C-V2X` 是设备/技术词，不得作为合同主体。

请求体：

```json
{
  "rules": [
    {
      "standardPartyName": "上汽帆一",
      "aliases": ["尚行", "上海帆一尚行科技有限公司", "帆一"]
    }
  ]
}
```


返回字段：

- id
- sourceRowNo
- packageName
- packageAmount
- chain

### 8.10 确认 / 否决前后向候选关系

```text
POST /api/v1/front-back-contract-candidates/{id}/decision
```

请求体：

```json
{
  "decision": "confirmed",
  "reviewComment": "主体、设备关键词、税率和付款条款一致，进入设备级拆分。"
}
```

`decision` 可选：`confirmed`、`rejected`、`candidate`。只有 `confirmed` 才允许进入设备级资金流单元生成流程。

```json
      "ownerAuditResults": [],
```

### 8.18 合同风险列表

```text
GET /api/v1/contract-risks
```

请求参数：

- pageNo
- pageSize
- riskType
- severity
- partyName
- nodeId

风险类型至少包括：

- unconfirmed_candidate_used
- front_confirmed_without_device_split
- back_contract_without_front_source
- duplicated_device_claim
- upstream_audited_not_received
- downstream_payment_without_owner_audit
- audit_amount_less_than_contract_amount
- tax_rate_mismatch
- downstream_over_released
