## 2. 点位管理接口

### 2.1 点位列表查询

```text
GET /api/v1/sites
```

用途：表格视图、卡片视图共用。点位管理页面不分页展示，前端使用右侧滚动条浏览；后端可按性能需要支持游标或滚动加载。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| nodeId | string | 否 | NodeID |
| siteName | string | 否 | 点位名称 |
| district | string | 否 | 行政区域 |
| originalDistrict | string | 否 | 原始行政区域，如经开 |
| siteType | string | 否 | 点位类型 |
| signalVendor | string | 否 | 信号机厂商 |
| perceptionSiteType | string | 否 | 感知点位类型 |
| isAdaptiveSite | boolean | 否 | 是否自适应点位 |
| isVariableLaneSite | boolean | 否 | 是否可变车道路口 |
| hasIssue | boolean | 否 | 是否有异常 |
| sortIssueFirst | boolean | 否 | 是否将异常点位排在前面，默认 true |

说明：

- 点位总数按有效点位行数统计，不按 Excel 物理行数统计。
- NodeID 为空、NodeID 重复或关键点位信息不足的导入行不得进入正式点位库，应进入导入错误行或待补全治理清单。
- 经开区在项目过程中已并入新吴区；接口筛选 `district` 使用归并后的标准行政区域，`originalDistrict` 仅用于审计和回查。

返回字段：

- id
- nodeId
- crossId
- siteName
- nodeNameEn
- district
- originalDistrict
- districtNormalizeRule
- siteType
- signalVendor
- perceptionSiteType
- isAdaptiveSite
- isVariableLaneSite
- longitudeGcj02
- latitudeGcj02
- deviceRecordCount
- deviceTypeCount
- installedQuantity
- issueCount
- updatedAt

### 2.1A 删除点位

```text
DELETE /api/v1/sites/{siteId}
```

用途：逻辑删除点位。

删除规则：

- 删除前必须校验点位下是否存在 matched `device_installation_item`。
- 若存在下挂设备，返回 `SITE_HAS_DEVICE_ITEMS`，提示用户先到设备管理删除或迁移该点位下设备。
- 删除成功时写入 `deleted_at`、`deleted_by`、`delete_reason`，并刷新周边点位缓存。

请求体：

```json
{
  "deleteReason": "重复点位清理"
}
```

返回字段：

- id
- deletedAt
- deletedBy

### 2.2 地图点位查询

```text
GET /api/v1/sites/map
```

用途：地图视图查询当前筛选范围点位。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| district | string | 否 | 行政区域 |
| siteType | string | 否 | 点位类型 |
| hasIssue | boolean | 否 | 是否有异常 |
| minLng | decimal | 否 | 当前地图视野最小经度 GCJ-02 |
| minLat | decimal | 否 | 当前地图视野最小纬度 GCJ-02 |
| maxLng | decimal | 否 | 当前地图视野最大经度 GCJ-02 |
| maxLat | decimal | 否 | 当前地图视野最大纬度 GCJ-02 |

返回字段：

- id
- nodeId
- siteName
- district
- siteType
- signalVendor
- perceptionSiteType
- isAdaptiveSite
- isVariableLaneSite
- longitudeGcj02
- latitudeGcj02
- deviceRecordCount
- deviceTypeCount
- installedQuantity
- issueCount

### 2.3 点位详情

```text
GET /api/v1/sites/{siteId}
```

用途：点位编辑页基础信息。

返回字段：

- site 全量字段
- statistics 统计字段

### 2.4 保存点位

```text
PUT /api/v1/sites/{siteId}
```

用途：保存点位编辑页基础信息。

请求体：

```json
{
  "siteType": "匝道汇入",
  "district": "锡山",
  "originalDistrict": "锡山",
  "nodeId": "10372",
  "crossId": "-1",
  "nodeNameEn": "TaiHuDaDaoBei~YouYiNanLuXi",
  "siteName": "太湖大道北~友谊南路西",
  "signalVendor": "华通",
  "perceptionSiteType": "雷视融合",
  "isAdaptiveSite": false,
  "isVariableLaneSite": false,
  "variableLaneCount": null,
  "variableLaneDirection": ""
}
```

规则：

- NodeID 必须唯一。
- 点位编辑页面默认不编辑或展示原始 CGCS2000 坐标；如需调整坐标，应通过坐标治理流程刷新 GCJ-02 坐标。
- GCJ-02 坐标变化后刷新周边点位缓存。

### 2.5 点位周边点位

```text
GET /api/v1/sites/{siteId}/neighbors
```

用途：点位编辑页嵌入式地图展示当前点位周边 4-5 个点位。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| limit | int | 否 | 默认 5 |

返回字段：

- rankNo
- siteId
- nodeId
- siteName
- district
- siteType
- longitudeGcj02
- latitudeGcj02
- distanceMeters

### 2.6 重新转换点位坐标

```text
POST /api/v1/sites/{siteId}/coordinate/convert
```

用途：手动重新执行 CGCS2000 到 GCJ-02 转换。

返回字段：

- longitudeGcj02
- latitudeGcj02
- coordinateConvertMethod
- coordinateConvertStatus
- coordinateConvertTime
- 原始 CGCS2000 坐标仅在审计日志中返回

### 2.7 坐标异常列表

```text
GET /api/v1/sites/coordinate-errors
```

用途：坐标异常处理页。

请求参数：

- pageNo
- pageSize
- nodeId
- siteName
- district
- coordinateStatus

返回字段：

- id
- nodeId
- siteName
- district
- longitudeGcj02
- latitudeGcj02
- coordinateConvertStatus
- errorMessage
- updatedAt

说明：坐标治理页面默认显示 GCJ-02 坐标；原始 CGCS2000 坐标仅作为审计字段按需展开。
