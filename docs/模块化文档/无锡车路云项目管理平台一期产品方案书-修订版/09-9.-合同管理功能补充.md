## 9. 合同管理功能补充

无锡车路云一体化项目合同结构复杂，总集单位为移动+天安。系统合同管理不能只做旧静态合同表，也不能按 Excel sheet 管理合同边界。新版合同管理应定位为：

```text
合同关系重构与设备级阶段资金流管理
```

### 9.1 宏观合同流骨架

系统应读取 `无锡车路云一体化采购价格测算表202500920.xlsx` 中 `合同流` sheet 尾部宏观合同流，识别移动、天安、浪潮、上研、中通服网盈、尚行、四维图新等资金骨架。

必须使用修正后的宏观合同流名称：

```text
移动-四维图新-天安（大华+海康智联+RSU)
```

不得使用旧的“吉利/希迪版本”表述。`RSU` 是设备，不是合同主体。

### 9.2 Word 前后向合同匹配

合同边界必须来自正式 Word 前向销售合同和后向采购合同文本。系统应抽取：

- 合同方向。
- 甲方、乙方、相对方。
- 合同金额。
- 税率。
- 设备/软件/服务关键词。
- 付款条款。
- 背靠背付款条款。
- 是否依赖业主/甲方客户回款。

### 9.3 前向合同与后向合同多对多关系

系统应生成并人工确认：

```text
前向合同 → 天安 → 后向合同
```

的多对多候选关系。一个前向合同可以拆给多个后向合同，多个前向合同也可能汇入一个后向合同。候选关系未确认前不得进入资金计算。

### 9.4 设备级资金流

合同级关系确认后，必须继续钻取到设备/软件/服务明细级：

```text
前向合同明细
→ 后向合同明细
→ Excel 清单池行
→ NodeID/点位/安装位置
→ 业主设备级审计
→ 上游回款
→ 后向付款释放
```

上游回款需要经过业主设备级跟踪审计。下游付款也必须根据设备级审计结果和上游回款结果释放，不能只看合同级付款比例。

### 9.5 阶段级部分付款

设备级资金流仍然不是一次性付款。系统应按阶段管理：

```text
预付款 / 到货款 / 安装款 / 验收款 / 审计款 / 质保金
```

每个阶段都要判断：

- 是否满足合同条款。
- 是否需要到货、安装、验收。
- 是否需要业主审计。
- 是否需要上游回款。
- 审计确认数量和金额是多少。
- 可释放后向付款金额是多少。

前向合同和后向合同的付款阶段可能不一致，不能机械一一对应。系统应支持：

```text
前向设备级阶段回款
→ 设备级资金池
→ 后向设备级阶段付款释放
```

### 9.6 合同风险

系统应识别：

- 前向合同已确认但未拆到设备。
- 后向合同找不到对应前向资金来源。
- 同一设备被多个后向合同重复认领。
- 前向已审计但未回款。
- 后向申请付款但对应设备未审计。
- 审计确认金额小于合同金额。
- 前后向税率不一致。
- 后向阶段付款超过设备级可释放金额。
