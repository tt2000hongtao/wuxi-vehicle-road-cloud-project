## 9. 合同管理页面

新版合同管理页面不再是静态旧静态合同表，也不再让用户按 Excel 清单工作表确认合同边界。页面定位为：

```text
合同关系重构工作台
+ 宏观合同流骨架
+ Word 合同解析复核
+ 前向合同 → 天安 → 后向合同多对多候选关系确认
+ 设备级资金流单元
+ 业主设备级审计跟踪
+ 阶段付款释放看板
```

### 9.1 合同关系重构工作台

页面顶部展示当前重构状态，而不是展示静态金额卡片：

- 宏观合同流数量。
- Word 合同数量。
- 前向合同数量。
- 后向合同数量。
- 前后向候选关系数量。
- 已确认、待确认、已否决数量。
- 设备级资金流单元生成状态。
- 高风险锁定项数量。

原型状态说明：确认结果暂存 `localStorage`，正式系统应写入后端审核表。

### 9.2 宏观合同流骨架

展示 Excel `合同流` 工作表尾部识别出的宏观资金骨架。该骨架用于理解资金链路，不用于直接生成合同边界。

交互要求：

- 默认选中第一条宏观合同流。
- 点击某条宏观合同流后，筛选相关 Word 合同和前后向候选关系。
- 可切回“全部宏观流”。
- 每条宏观流展示：Excel 行号、标准合同流名称、主体链路、天安侧金额、链路类型。

必须使用修正名称：

```text
移动-四维图新-天安（大华+海康智联+RSU)
```

其中 `RSU` 是设备，不是合同主体。

### 9.3 Word 合同解析复核

Word 合同文本决定合同边界。列表展示每份合同的程序化解析结果：

- 合同方向：前向销售 / 后向采购。
- 合同文件名。
- 主体链。
- 合同金额线索。
- 税率线索。
- 设备/服务关键词。
- 匹配到的宏观合同流。
- 解析可信度和待复核状态。

用户需要在正式系统中确认：方向、主体、金额、税率、付款条款和归属宏观流是否正确。

### 9.4 前向合同 → 天安 → 后向合同候选关系

这是页面核心工作区。每行表示一个候选关系：

```text
某前向销售合同
→ 经天安
→ 某后向采购合同
```

候选行必须展示：

- 置信度。
- 前向合同名称。
- 后向合同名称。
- 共同宏观合同流数量。
- 税率重合情况。
- 主体、设备/服务关键词、付款条款等证据。
- 状态：待确认、已确认、已否决。

交互要求：

- 按宏观合同流筛选。
- 按确认状态筛选。
- 按置信度筛选。
- 按税率筛选。
- 按合同名、主体或设备关键词搜索。
- 查看证据明细。
- 对候选关系执行“确认”“否决”“恢复为候选”。

候选关系未确认前，绝不能进入应收、应付、毛利或付款释放计算。

### 9.5 设备级资金流单元

合同级关系确认后，继续拆到设备/软件/服务项和点位级。最小管理单元为：

```text
前向合同明细
→ 后向合同明细
→ 清单池行
→ NodeID / 点位 / 安装位置
→ 业主设备级审计
→ 上游阶段回款
→ 后向阶段付款释放
```

页面应展示：

- 设备/服务名称。
- 规格型号。
- NodeID 和点位。
- 前向合同、金额、税率。
- 后向合同、金额、税率。
- 数量、价差、毛利。
- 审计状态。
- 当前回款阶段。
- 当前付款阶段。
- 锁定原因和风险等级。

### 9.6 业主设备级审计跟踪

上游回款不是合同整体回款判断，而是设备级审计结果驱动。

必须支持状态：

- 待审计。
- 已提交。
- 审计通过。
- 部分通过。
- 审计不通过。
- 需补资料。

只有通过业主设备级审计并满足合同条款的设备，才可以进入对应阶段的上游回款判断。

### 9.7 阶段付款释放看板

设备级资金流仍需按阶段部分释放，至少包含：

```text
预付款 / 到货款 / 安装款 / 验收款 / 审计款 / 质保金
```

每个阶段展示：

- 触发条件。
- 业主审计要求。
- 上游回款要求。
- 后向付款释放规则。
- 是否允许部分释放。
- 是否被审计结果锁定。
- 是否存在上游未回款但下游付款风险。

### 9.8 点位/设备穿透

合同管理页面应能向下钻取到：

- 点位。
- NodeID。
- 设备安装记录。
- 验收资料。
- 审计结果。
- 出入库单据。
- 合同原文和附件。
- 付款记录。

点位页和设备页展示的合同信息，应来自已确认的设备级资金流单元，而不是来自清单工作表名称。
