## 12. 设备级合同履约、资金流与付款

### 12.1 设备级履约状态

合同履约不只看合同状态，还要看设备级过程：

```text
合同关系确认
→ 清单池行拆分
→ NodeID / 点位绑定
→ 到货 / 入库 / 领料
→ 安装 / 验收
→ 业主设备级审计
→ 上游阶段回款
→ 后向阶段付款释放
→ 质保闭环
```

同一合同中的不同设备，可能分批到货、分批安装、分批验收、分批审计、分批付款。因此系统必须在设备级记录每一个节点的发生时间和证据文档。

### 12.2 设备级资金池

资金流动必须细化到设备级资金流单元。系统需要同时管理：

```text
业主/车联网集团 → 移动/浪潮/上汽帆一等前向主体 → 天安
天安 → 后向供应商/施工单位/服务商
```

设备级资金字段包括：

- 前向合同、后向合同。
- 前向含税金额、税率、不含税金额。
- 后向含税金额、税率、不含税金额。
- 设备数量、NodeID、点位。
- 业主审计确认数量和金额。
- 上游已回款金额。
- 后向已付款金额。
- 当前可付款金额。
- 是否超审计、超阶段、超资金池。

### 12.3 阶段付款

设备级仍然需要按阶段部分释放：

- 预付款。
- 到货款。
- 安装款。
- 验收款。
- 审计款。
- 质保金。

每个阶段都要判断：

- 合同条款是否满足。
- 是否需要到货、安装、验收。
- 是否需要业主设备级审计。
- 是否需要上游回款。
- 审计确认数量和金额是多少。
- 后向可付款金额是多少。

### 12.4 背靠背付款规则

后向采购合同如存在背靠背付款逻辑，后向付款必须绑定上游回款和设备级审计结果。

系统应支持：

- 付款阶段绑定前向回款阶段。
- 若上游未回款，后向付款阶段自动锁定。
- 若设备未通过业主审计，后向付款阶段自动锁定。
- 若上游部分回款，按设备级资金池和审计确认金额分摊可付款额度。
- 前向阶段和后向阶段可不一致，不允许机械一一对应。
