## 12. 设备级合同履约、资金流与付款

### 12.1 设备级履约状态

合同履约不只看合同状态，还要看设备级过程：

```text
已签约
→ 已下单
→ 已送货
→ 已入库
→ 已开箱验收
→ 已领料
→ 已安装
→ 已点位验收
→ 已付款
→ 已归档
```

同一合同中的不同设备，可能分批到货、分批入库、分批安装、分批验收，也可能触发不同批次付款。因此系统必须在设备级记录每一个节点的发生时间。

设备级履约事件：

- 合同明细确认
- 下单
- 送货
- 到货查验
- 入库
- 开箱验收
- 领料
- 安装
- 点位初验
- 试运行
- 终验
- 审计确认
- 质保开始
- 质保结束

### 12.2 资金流动

资金流动必须细化到设备/合同明细级。系统需要同时管理：

```text
业主/车联网集团 → 移动/浪潮/上汽帆一等前向主体 → 天安
天安 → 后向供应商/施工单位/服务商
```

资金流字段：

- 资金方向：收入、支出、回流、代付、暂估、质保金。
- 关联设备/合同明细。
- 关联前向合同。
- 关联后向合同。
- 付款触发事件。
- 应收金额、实收金额、应付金额、实付金额。
- 上游回款比例。
- 后向付款比例。
- 是否背靠背付款。
- 是否需要回流业主车联网集团。

天安自有或集成费处理规则：

- 不能简单标记为 `self_owned` 后结束。
- 其中一部分需要通过采购数据、设备清单、服务确认或审计结果回流到业主车联网集团。
- 系统需标记 `owner_backflow_required`，并追踪回流依据、回流金额、回流状态和对应业主确认材料。

### 12.3 付款节点

付款节点支持按合同、分项包、设备明细维度绑定：

- 合同签署款
- 到货款
- 入库验收款
- 安装完成款
- 初验款
- 终验款
- 质保金

付款触发条件可配置：

- 合同签署完成
- 到货数量达到合同数量比例
- 入库数量达到合同数量比例
- 安装数量达到合同数量比例
- 点位验收通过
- 发票已到
- 上游回款已到账
- 业主或甲方客户已完成同阶段验收

### 12.4 背靠背付款规则

后向采购合同普遍存在背靠背付款逻辑：甲方向乙方付款以业主或甲方客户向甲方支付相应等比例款项为前提。

系统应支持：

- 付款节点绑定上游回款节点。
- 若上游未回款，后向付款节点自动标记为“待上游回款”。
- 若设备已验收但上游未回款，形成资金风险。
- 若上游部分回款，按设备/明细级分摊可付款额度。
