## 2. 核心结论

合同管理模块不能再按普通旧静态合同表、静态合同流图或 Excel sheet 明细台账设计。新版模块应定义为：

```text
合同关系重构与设备级阶段资金流管理
```

正确业务主线是：

```text
Excel 宏观合同流骨架
→ Word 前向/后向合同文本边界
→ 前向合同 → 天安 → 后向合同多对多候选关系
→ 人工确认/否决
→ 设备级资金流单元
→ NodeID / 点位 / 安装位置
→ 业主设备级审计
→ 上游阶段回款
→ 后向阶段付款释放
```

核心规则：

- Excel 各清单工作表只是设备、软件、服务清单池，不是合同边界。
- 合同边界必须来自 Word 前向销售合同和后向采购合同文本。
- 前向合同与后向合同是经天安形成的多对多关系。
- 税率是一等匹配字段，必须进入合同、候选关系、设备级资金单元和付款记录校验。
- 候选关系未经人工确认，`financialUseAllowed` 必须为 `false`。
- 上游回款与下游付款都必须受业主设备级审计结果和阶段付款条件约束。
- 设备级仍可能按预付、到货、安装、验收、审计、质保分阶段部分付款。

系统必须支持从任意对象双向追溯：

```text
点位 → 设备 → 设备级资金流单元 → 前向合同/后向合同 → 宏观合同流 → 业主审计 → 阶段付款

合同 → 候选关系 → 已确认关系 → 设备/服务清单池行 → NodeID → 审计批次 → 可回款/可付款金额
```
