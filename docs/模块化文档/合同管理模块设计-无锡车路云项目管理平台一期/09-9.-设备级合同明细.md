## 9. 设备级合同明细

### 9.1 明细来源

各分项 sheet 中常见字段包括：

- 序号
- 分项名称
- 性能指标参数
- 品牌及型号
- 所属系统及部署位置
- 单位
- 数量
- 单价
- 金额
- 税点/税率
- 备注/用途
- 合同
- 原厂单价
- 原厂采购价格
- 毛利

软件和服务类 sheet 则包含：

- 子系统名称
- 一级模块
- 二级模块
- 工作量
- 综合单价
- 金额
- 税率
- 备注

### 9.2 合同明细类型

| 类型 | 说明 |
|---|---|
| hardware_device | 硬件设备 |
| software_product | 软件产品 |
| custom_development | 定制开发 |
| service | 运维、租赁、测评、安装等服务 |
| construction | 施工或安装工程 |
| network_resource | 物联网卡、骨干光纤等资源 |
| placeholder | 暂时不实施或预留项 |

### 9.3 设备级管理要求

设备级合同明细必须支持：

- 记录合同链路。
- 记录每一条设备/服务明细对应的前向合同主体、后向合同主体、最终供应商/原厂。
- 记录投标价、采购价、原厂价、毛利。
- 关联设备安装位置表中的 `物料名称`、`品牌型号`、`NodeID`、`安装位置`。
- 关联点位。
- 关联入库、出库、领料、安装、验收数量。
- 识别合同数量与实际设备数量差异。
- 支持同一个分项 sheet 内不同设备分属不同前向/后向合同主体。
- 支持同一合同内不同设备的到货、入库、安装、验收、付款节点发生时间不同。

### 9.4 设备级前后向合同归属

各分项 sheet 不能只按 sheet 名或分项包统一确定合同主体。真实情况是：同一个子模块、同一个分项 sheet 下，不同设备可能对应不同的前向销售合同、后向采购合同、原厂供应商和资金路径。

因此合同明细必须采用“行级合同归属”：

```text
分项 sheet 行
→ 合同明细 contract_item
→ 前向合同归属 contract_item_contract_role(role=front_sales)
→ 后向合同归属 contract_item_contract_role(role=back_purchase)
→ 原厂/最终供应 contract_item_contract_role(role=original_supplier)
```

字段要求：

- 前向合同主体：如浪潮、上汽帆一、中通服网盈、工业安装等。
- 后向合同主体：如万集、航天海特、华通、海康智联等。
- 天安角色：销售方、采购方、集成方、自有交付方。
- 合同文件：关联前项销售 PDF 或后项采购 PDF。
- 合同金额分摊：支持按数量、按金额、按人工确认分摊。
- 设备范围：支持关联多个点位、多个安装位置、多个 SN。
