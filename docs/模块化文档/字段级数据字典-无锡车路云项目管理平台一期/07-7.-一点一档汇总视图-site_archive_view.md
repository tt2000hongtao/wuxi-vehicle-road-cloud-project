## 7. 一点一档汇总视图 `site_archive_view`

一点一档建议作为汇总视图或接口聚合模型，不一定落单表。

| 字段组 | 字段内容 | 来源 |
|---|---|---|
| 基础档案 | NodeID、CrossID、点位名称、点位类型、行政区域、原始行政区域、信号机厂商、感知点位类型、是否自适应点位、是否可变车道路口 | site |
| 坐标地图 | GCJ-02 经纬度、周边点位；原始 CGCS2000 经纬度仅审计 | site、site_neighbor_cache |
| 信号系统 | 信号机厂商、自适应点位、可变车道 | site |
| 验收单 | 路口编号、管理部门、验收单经纬度、路口类型、方向道路、设备验收明细、功能测试结果 | site_acceptance_form、site_acceptance_device_item、site_acceptance_test_item |
| 接线表 | 施工单位、信号机品牌、信号灯位置、信号灯类别、数量、接线位置、端子号 | signal_wiring_form、signal_wiring_item |
| 点位照片 | 点位全景、设备照片、封面图 | site_archive_photo、document_file |
| 设备清单 | 按设备类型逐项展示安装位置、物料编码、物料名称、供应商、合同、数量、状态 | device_installation_item |
| 单据资料 | 送货单、入库单、出库单、开箱验收、领料单、序列号验证 | device_installation_item、document_file |
| 问题异常 | 开箱验收问题、工单、施工整改问题 | device_installation_item、work_order、rectification_issue |
| 运维记录 | 工单、维修、更换、巡检 | work_order、inspection_record |

自动生成规则：

1. 系统扫描一路一档资料目录，按文件夹名称生成 `site_archive_folder`。
2. 通过文件夹名称、验收单点位名称、接线表点位名称与 `site.site_name` 进行标准化匹配。
3. 匹配成功后自动解析 `验收单.docx`、`接线表.xls` 和点位照片。
4. 解析失败的字段保留原始文件，并在 `parse_message` 中记录。
5. 一点一档页面实时聚合结构化数据和附件资料，不要求人工重复录入。
