## 8. 合同管理数据结构

合同管理数据结构已按“合同关系重构与设备级阶段资金流”重新定义。Excel `合同流` sheet 只提供宏观资金骨架，各清单工作表只作为设备/软件/服务清单池；合同边界必须来自 Word 前向销售合同和后向采购合同文本。未经人工确认的前后向候选关系，不得进入应收、应付、毛利或付款释放计算。

### 8.1 合同主体表 `contract_party`

| 字段名 | 中文名 | 类型 | 必填 | 来源 | 说明 |
|---|---|---:|---|---|---|
| id | 系统主键 | bigint | 是 | 系统生成 |  |
| party_name | 单位名称 | varchar(200) | 是 | 合同文本/宏观合同流 | 如移动、天安、浪潮、四维图新、海康智联 |
| standard_party_name | 标准单位名称 | varchar(200) | 是 | 系统/人工 | 用于归一统计 |
| original_party_name | 原始单位名称 | varchar(200) | 否 | 导入/解析 | 保留合同或 Excel 原文 |
| party_short_name | 单位简称 | varchar(100) | 否 | 系统/人工 | 用于页面展示 |
| party_type | 单位类型 | varchar(50) | 否 | 人工/规则 | 业主、总集成员、前向客户、后向供应商、原厂、服务商 |
| is_consortium_member | 是否联合体成员 | boolean | 是 | 系统/人工 | 移动、天安为是 |
| alias_names | 别名 | varchar(500) | 否 | 系统/人工 | 如尚行、上海帆一尚行、帆一 |
| remark | 备注 | text | 否 | 人工 |  |
