# 文档资产管理模块设计：无锡车路云项目管理平台一期

## 1. 设计目标

文档资产管理模块用于管理合同、会议纪要、变更单、项目计划、出入库单据、安装签证、验收资料、运维资料、地图配置资产等多格式文件。目标不是建设普通网盘，而是建设可被项目管理、合同履约、点位建设、设备治理、地图质检和运维闭环共同使用的文档资产底座。

核心原则：

- 文件原件进入统一文档资产库，前端不内嵌大文件。
- 数据库保存结构化元数据、版本、关联关系、解析结果、权限和审计，不保存大文件内容。
- 所有文件通过 `storage_key` 引用，兼容本地文件库、NAS、MinIO、S3、OSS、COS。
- 文档必须关联到项目业务对象，支持从点位、设备、合同、地图资产、施工任务、出入库记录、变更事项、会议、工单反向追溯。
- 地图 MAP/RSI/信号机文件继续按配置资产治理，作为专业资产类型挂接到文档资产体系。

## 2. 资产分层

```text
文档原件存储层
→ 文档元数据层
→ 解析与质检层
→ 业务对象关联层
→ 搜索、预览、下载和审计服务
→ 前端文档资产中心与对象详情页
```

### 2.1 文档原件存储层

本地部署阶段使用 `document_assets/raw` 或 NAS 存放原件；云端迁移阶段切换为对象存储。应用层不得依赖本机绝对路径，应通过统一 `storage_key` 和存储适配器访问。

建议目录键：

```text
raw/{project_code}/{document_category}/{yyyy}/{sha256_prefix}/{sha256}/{file_name}
```

### 2.2 元数据层

每个文件至少登记：

- 文件名、扩展名、MIME 类型、文件大小、`sha256`。
- 所属项目、文档分类、来源批次、来源路径或来源系统。
- 当前版本、审核状态、归档状态、上传人、上传时间。
- `storage_provider`、`storage_bucket`、`storage_key`。

### 2.3 业务关联层

文档关系不限制为一个对象。一个文档可同时关联合同、点位、设备安装项和出入库单；一个业务对象可挂接多份文档。

对象类型统一枚举：

| object_type | 说明 |
|---|---|
| site | 点位 |
| device | 设备安装项 |
| contract | 合同 |
| contract_item | 合同明细 |
| contract_flow_line | 合同流链路 |
| map_asset | 地图配置资产 |
| construction_task | 施工任务 |
| warehouse_record | 出入库记录 |
| change_record | 变更事项 |
| meeting | 会议纪要 |
| work_order | 运维工单 |

## 3. 文档分类

| 大类 | 子类示例 | 优先关联对象 |
|---|---|---|
| 合同类 | 总集合同、销售合同、采购合同、补充协议、付款申请、发票资料 | contract、contract_item、contract_flow_line |
| 会议类 | 会议纪要、周例会、专题会、纪要确认单 | meeting、change_record、site |
| 变更类 | 设计变更、工程变更、需求变更、价格变更、签证变更 | change_record、site、contract |
| 计划类 | 项目计划、施工计划、进度计划、交付计划 | construction_task、site |
| 物资类 | 出入库单、到货单、领料单、设备清单 | warehouse_record、device、contract_item |
| 施工类 | 安装签证、施工记录、照片、接线表、验收单 | site、device、construction_task |
| 地图类 | MAP、RSI、信号机配置、SVG 预览、质检报告 | map_asset、site |
| 运维类 | 工单、巡检记录、故障报告、整改闭环资料 | work_order、site、device |
| 管理类 | 汇报材料、审批单、内部说明、风险清单 | project、meeting |

## 4. 功能设计

### 4.1 文档资产中心

提供统一列表、分类筛选、对象类型筛选、状态筛选、文件名搜索、来源批次搜索、重复文件提示和未关联文档治理。

列表核心字段：

- 文档名称
- 分类
- 文件类型
- 文件大小
- 版本
- 审核状态
- 关联对象
- 来源批次
- 上传时间
- 解析状态
- 质检状态

### 4.2 对象资料页

在点位、合同、设备、地图资产、出入库、运维等页面提供“资料”页签，通过对象接口读取关联文档，不加载大文件原件。

典型穿透路径：

```text
点位 → 设备 → 安装签证 → 验收单 → 照片 → 运维工单
合同 → 分项包 → 设备明细 → 出入库单 → 安装记录 → 付款节点
地图资产 → MAP/RSI/信号机配置 → 版本 → 质检结果 → 对应点位
变更单 → 受影响点位 → 受影响合同 → 受影响设备 → 关联会议纪要
```

### 4.3 批量导入

批量导入分为扫描、登记、复制或挂载、解析、质检、确认六步。每批生成导入报告，记录文件数量、总大小、重复文件数、解析成功/失败数、未关联对象文件数。

### 4.4 解析与质检

第一阶段先做轻量解析：

- PDF：文件名、页数、文本摘要、疑似盖章页、主体/金额/日期关键字线索。
- DOCX：正文、表格、批注、修订痕迹线索。
- XLSX：工作表、表头、行数、关键字段。
- 图片：尺寸、EXIF、预览。
- XML/JSON/SVG：地图配置资产引用现有 MAP/RSI 解析与 SVG 预览规则。

质检规则：

- `sha256` 重复。
- 缺少业务对象关联。
- 合同金额与合同流台账不一致。
- 安装签证点位不存在。
- 出入库设备无法匹配合同明细。
- 地图文件版本滞后或缺少 MAP/RSI/信号机配置。
- 同一 NodeID 存在多份冲突验收资料。

## 5. 迁移策略

- 不一次性搬迁几十 GB 文件，先试点合同、点位一档资料、地图配置资产。
- 原始来源文件不删除；新系统先复制或登记文件，再计算哈希并建立元数据。
- 本地库与对象存储共用同一套 `storage_key` 语义。
- 后续云迁移只替换存储适配器，不重写业务模块和前端页面。

## 6. 一期落地边界

一期只要求完成：

1. 文档资产目录规范。
2. 文档元数据 ER 与接口定义。
3. 只读扫描脚本和扫描报告。
4. 前端“文档资产中心”原型，展示元数据、治理指标和对象关联路径。
5. 支持合同、点位一档资料、地图配置资产三类试点接入。

暂不做复杂 RAG 知识库、跨项目权限矩阵、OCR 全自动抽取和云端生产部署。
