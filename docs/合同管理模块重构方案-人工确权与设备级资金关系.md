# 合同管理模块重构方案：人工确权与设备级资金关系

> 日期：2026-06-06  
> 状态：替代原“程序自动重建合同流关系”的主流程设计  
> 适用项目：无锡市车路云一体化项目管理平台  

## 1. 重构结论

当前合同流是人为切割、分包、转包、回流后形成的复杂资金与履约网络。仅依靠程序从合同文件和 Excel 中自动筛查、自动理解合同流向，风险过高。

新的合同管理模块不再以“自动理解合同流”为主流程，而改为：

```text
人工确权合同归属
  +
程序辅助抽取合同文件和清单
  +
人工确认前后向清单关联
  +
设备级审计、收款、付款、利润留存跟踪
```

程序和 AI 的角色降级为：

- 辅助导入
- 辅助抽取
- 辅助比对
- 辅助发现异常
- 辅助生成候选关系

但合同流归属、合同金额、清单税率、前后向关联关系，必须由业务人员确认后才能生效。

## 2. 旧方案废弃点

废弃以下主流程假设：

1. 程序可通过合同文件名、正文、金额、关键词自动判断合同流归属。
2. 程序可自动判断一个前向合同应流向哪些后向合同。
3. 程序可直接用合同文本中抽出的金额汇总作为合同管理金额。
4. 程序可用设备关键词决定前向资金入口。
5. 自动匹配结果可直接进入资金流统计。

保留但降级：

- 自动扫描合同文件。
- 自动提取合同正文、表格、金额、税率候选。
- 自动生成前后向清单候选匹配。
- 自动提示税率、金额、数量、设备名称、规格型号差异。

## 3. 新主流程

### 3.1 梳理前向销售合同

操作路径：

```text
导入前向销售合同
  → 人工确定所属合同流链路
  → 人工确认合同金额
  → 导入/确认软硬件和服务清单
  → 确认清单每行税率
  → 生效为前向合同资金与履约基础数据
```

支持方式：

- 单独导入一个合同。
- 批量导入多个合同。
- 从 Word/PDF/Excel 附件中抽取清单候选。
- 从 Excel 设备清单 sheet 中选择或粘贴清单。
- 人工编辑和确认每一行。

前向合同必须确认字段：

| 字段 | 说明 |
|---|---|
| 合同文件 | 原件或扫描件 |
| 合同名称 | 人工确认 |
| 所属合同流链路 | 人工选择，不由程序最终决定 |
| 合同金额 | 人工确认，记录来源 |
| 合同角色 | 总合同 / 天安侧前向入口 / 补充协议 / 变更协议 |
| 上级总合同 | 如来自联合体总合同，应挂接总合同 |
| 清单行 | 软硬件、服务、平台、施工、运维等分项 |
| 每行税率 | 只允许 13%、9%、6% |
| 状态 | 草稿、待确认、已确认、已作废 |

### 3.2 梳理后向采购合同

操作路径：

```text
导入后向采购合同
  → 人工确定所属合同流链路
  → 人工选择关联的一个或多个前向销售合同
  → 人工确认合同金额
  → 导入/确认软硬件和服务清单
  → 确认清单每行税率
  → 生效为后向采购资金与履约基础数据
```

后向合同必须确认字段：

| 字段 | 说明 |
|---|---|
| 合同文件 | 原件或扫描件 |
| 合同名称 | 人工确认 |
| 供应商/乙方 | 人工确认 |
| 所属合同流链路 | 人工选择 |
| 关联前向销售合同 | 可多选 |
| 合同金额 | 人工确认，记录来源 |
| 清单行 | 后向采购设备/服务明细 |
| 每行税率 | 只允许 13%、9%、6% |
| 付款条款 | 阶段、比例、条件 |
| 状态 | 草稿、待确认、已确认、已作废 |

### 3.3 设备级/清单行级关联

系统核心工作台应从“合同级匹配”改为“清单行级匹配”：

```text
前向合同清单行
  ↔
后向采购合同清单行
  ↔
设备/服务标准项
  ↔
NodeID/点位
  ↔
业主审计结果
  ↔
前向收款
  ↔
后向付款
  ↔
利润留存
```

匹配对象不是“合同对合同”，而是：

```text
前向销售合同清单第 N 行 ↔ 后向采购合同清单第 M 行
```

每条关联必须支持：

- 一对一
- 一对多
- 多对一
- 多对多拆分
- 数量部分匹配
- 金额部分匹配
- 税率一致/不一致提示
- 按点位/NodeID 继续拆分

### 3.4 审计、收款、付款、利润留存

清单行关联确认后，形成设备级资金关系单元：

| 单元 | 说明 |
|---|---|
| 前向清单金额 | 应收侧金额 |
| 后向清单金额 | 应付侧金额 |
| 关联数量 | 可小于合同清单总数量 |
| 关联点位 | NodeID/点位，可为空但不能进入付款释放 |
| 审计确认数量 | 业主设备级审计确认 |
| 审计确认金额 | 业主确认金额 |
| 前向已收款 | 按阶段归集 |
| 后向可付款 | 受前向回款和审计结果约束 |
| 后向已付款 | 实际付款 |
| 利润留存 | 前向确认金额 - 后向确认/支付金额 |

付款必须支持阶段：

- 预付款
- 到货款
- 安装款
- 验收款
- 审计款
- 质保金

每阶段支持部分释放。

## 4. 页面重新设计

### 4.1 合同管理首页

不再以静态卡片为主，而以工作流为主：

```text
合同管理首页
  ├─ 前向销售合同梳理
  ├─ 后向采购合同梳理
  ├─ 前后向清单关联工作台
  ├─ 设备级审计与收付款台账
  ├─ 收支与利润留存看板
  └─ 异常与待确认事项
```

### 4.2 前向销售合同梳理页

功能：

- 上传合同文件。
- 批量导入合同文件。
- 查看合同原件。
- 人工选择合同流链路。
- 人工确认合同金额。
- 导入/编辑清单行。
- 确认每行税率。
- 提交为“已确认前向合同”。

### 4.3 后向采购合同梳理页

功能：

- 上传后向采购合同。
- 批量导入合同文件。
- 人工选择合同流链路。
- 人工关联一个或多个前向销售合同。
- 人工确认合同金额。
- 导入/编辑后向清单行。
- 确认每行税率。
- 提交为“已确认后向合同”。

### 4.4 前后向清单关联工作台

核心交互：

```text
左侧：前向合同清单行
右侧：后向采购合同清单行
中间：关联、拆分、确认、否决
底部：金额、税率、数量、利润、审计状态
```

支持：

- 按合同流链路筛选。
- 按前向合同筛选。
- 按后向合同筛选。
- 按设备/服务关键词筛选。
- 按税率筛选。
- 按未关联/已关联/异常筛选。
- 批量确认候选。
- 人工拆分数量和金额。

### 4.5 设备级审计与收付款台账

围绕确认后的清单关联生成台账：

- 按合同流链路看。
- 按前向合同看。
- 按后向合同看。
- 按设备/服务项看。
- 按点位/NodeID 看。
- 按审计阶段看。
- 按付款阶段看。

### 4.6 收支与利润留存看板

看板指标：

- 前向合同金额。
- 后向合同金额。
- 已确认前向清单金额。
- 已确认后向清单金额。
- 已审计金额。
- 已收款金额。
- 可付款金额。
- 已付款金额。
- 利润留存。
- 异常金额。

## 6. 程序能力重新定位

程序不再直接“判定合同关系”，而是提供候选和校验：

| 能力 | 是否可自动 | 是否需人工确认 |
|---|---|---|
| 扫描合同文件 | 是 | 否 |
| 提取合同正文 | 是 | 否 |
| 提取合同表格候选 | 是 | 是 |
| 判断合同流链路 | 仅候选 | 是 |
| 判断合同金额 | 仅候选 | 是 |
| 判断清单行税率 | 可候选 | 是 |
| 前后向清单匹配 | 仅候选 | 是 |
| 设备级收付款关系 | 不自动生效 | 是 |
| 利润留存统计 | 基于已确认数据自动 | 数据需确认 |

## 7. 当前原型改造优先级

第一阶段先做人工确权闭环：

1. 推翻现有合同管理首页静态卡片结构。
2. 新增“前向销售合同梳理”页面。
3. 新增“后向采购合同梳理”页面。
4. 新增“清单行关联工作台”。
5. 将自动分析结果全部标记为“候选”，不得直接作为已确认数据。
6. 新增人工确认状态字段。
7. 新增税率三选一控件：13%、9%、6%。
8. 新增合同流链路人工选择控件。
9. 新增前后向合同多选关联控件。
10. 新增分项收支和利润留存汇总。

## 8. 当前明确废弃的设计口径

以下口径废弃：

- “系统自动从合同和 Excel 完全理解合同流”。
- “前向合同自动归属合同流后即可进入资金统计”。
- “后向合同自动匹配前向合同后即可进入付款计算”。
- “合同文本抽取的金额自动作为合同金额”。
- “关键词匹配结果作为合同主要内容或合同流依据”。

新口径：

```text
人工确权是主流程，程序抽取是辅助能力。
```
